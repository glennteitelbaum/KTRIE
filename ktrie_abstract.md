# KTRIE: A Trie/B-Tree Hybrid for Ordered Associative Containers

## Abstract

We present the KTRIE, a cross between a trie and a B-tree that combines prefix compression, bitmap-indexed byte-level branch nodes, and compact sorted leaves to provide an ordered associative container with competitive lookup performance, efficient ordered iteration, and low memory overhead. The structure uses trie-style prefix routing to navigate to the right region of the key space, then stores remaining suffixes in B-tree-style wide sorted leaves. Each key is decomposed into three logical regions: a shared prefix absorbed into a single node, one or more branch levels that fan out only where the data requires, and a suffix region collected into a flat sorted array within a compact leaf. This decomposition adapts depth and memory footprint to the actual key distribution rather than fixing it to the key width.

The KTRIE offers O(K) lookup, insertion, and deletion, where K is the key length in bytes (a constant for any given instantiation). Its distinguishing contribution is suffix compression: the KTRIE truncates at a configurable depth, collecting remaining suffixes into flat sorted leaves. Where a traditional trie scatters sparse keys across many small deep nodes, suffix compression collapses them into contiguous arrays, eliminating per-node overhead, improving locality, and enabling O(1) amortized per-element ordered iteration.

We evaluate two concrete instantiations. KNTRIE is specialized for fixed-width integer keys (16-, 32-, and 64-bit). KSTRIE extends the design to variable-length string keys, addressing the challenges of unbounded key length and unpredictable trie depth. Both instantiations share a common value storage layer specialized by type: packed bits for booleans similar to `std::vector<bool>`, inline embedding for small trivial types (≤ 8 bytes) similar to SSO for `std::string`, and heap-indirected owned copies for larger or non-trivial types.

The implementations employ bitmap-compressed 256-way branch nodes with branchless popcount dispatch, and compact sorted leaves with binary search, both driven by compile-time template specialization. KNTRIE capitalizes on fixed key width to use depth and narrowing of keys. KSTRIE extends prefix compression into the suffix leaves themselves. Both use the compact leaf nodes where they fit. KNTRIE based on entry count, KSTRIE based on the compressed suffix key length. Each are tracked per bitmask node tracking of child summations.

Benchmarks demonstrate that both KNTRIE and KSTRIE are competitive with `std::unordered_map` across read, iteration, insert, and erase operations while providing full ordering, and consistently outperform `std::map` across all operations. By exploiting prefix compression and suffix truncation within compact leaves, a KTRIE can occupy less space than the raw key-value data (the sum of bare key and value sizes with no container overhead). Memory overhead per entry, relative to the raw key-value size, averages near zero and frequently falls below zero for key distributions with shared prefixes or compressible suffixes. Storage within leaves is contiguous, further improving cache locality.

The design draws on established techniques: the trie structure originates with Fredkin (1960) and is treated extensively by Knuth (1973, 1998); the B-tree was introduced by Bayer and McCreight (1972); bitmap-indexed child compression follows the approach of Judy arrays (Baskins, 2004) and the Adaptive Radix Tree (Leis et al., 2013); and the branchless binary search within leaves adapts the technique of Khuong and Morin (2017). The contribution of this work is the integration of these techniques into a trie/B-tree hybrid, combining trie-style prefix routing with B-tree-style wide sorted leaves and the prefix/branch/suffix decomposition, yielding an ordered, cache-local, memory-compressing container without the depth penalty of balanced trees, and with full ordering unlike hash tables. The keysuffix sharing invariant and the byte-budget collapse heuristic are, to our knowledge, novel contributions specific to this design.

**Keywords:** trie, B-tree, ordered associative container, prefix/suffix decomposition, keysuffix sharing, cache locality, bitmap indexing, branchless search, variable-length keys
